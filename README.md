# こころの立て直しワークブック PWA実験版

このリポジトリは、既存の安定版 `/home/chanmi/projects/mental-care-workbook` とは別に作った PWA 実験版です。安定版の教材内容や保存構造を壊さず、スマホのホーム画面追加や Cloudflare Pages 配信を試すための作業場所です。

## 目的

メンタルケア教材（全8種）を、利用者自身が選んで使えるラッパー型ワークブックアプリです。

それぞれのワークは、困った場面の整理、つらさへの対処の確認、行動指針の探索など、目的に応じて選んで使います。

- 評価・査定・監視のための道具ではありません。
- 空欄のままでも、途中でやめても大丈夫です。
- 書いた内容はこの端末の中だけに残ります。外部に送信されることはありません。

## 含まれるワーク

| # | ワーク | 内容 |
|---|--------|------|
| 1 | 問題を深く知る | 困った場面を整理し、何が起きているのかを見える化する |
| 2 | DOTS — つらさへの対処を知る | これまで使ってきた対処の工夫を振り返る |
| 3 | 創造的絶望 | コントロールをめぐる体験エクササイズ |
| 4 | 視覚化ツール | 頭の中を「見える形」にして整理する |
| 5 | 心のコンパス | 生活領域ごとに大切にしたいふるまいを考える |
| 6 | Leaves on the Stream | 頭に浮かぶ言葉を葉に乗せて流すワーク |
| 7 | 少し距離を取る | 「通知」と距離を取って小さな行動を決める |
| 8 | いまどうするか | 気がかりを整理し、今できる一歩を見つける |

## 起動方法

```bash
# 開発サーバー
npm install
npm run dev

# プロダクションビルド
npm run build
npm run preview

# 画面の見た目チェック
npm run visual-check
```

## PWA実験版としての方針

- 入力内容はサーバーに保存しません。ブラウザの端末内保存を使います。
- 別端末へ移すときは、表紙の「全体バックアップを保存」からバックアップファイルを保存し、移行先で読み込みます。
- Service Worker と manifest を追加し、ホーム画面追加と基本的なキャッシュ動作を試せる状態にしています。
- 教材本文、設問文、保存キー、全体バックアップ構造、ワーク6 Leaves の物理挙動は、このPWA化では原則変更しません。

## Cloudflare Pages 試験設定

Cloudflare 側のアカウント、Pages、Access 設定はこのリポジトリではまだ実行していません。公開する場合は、GitHub の公開リポジトリを作成してから Cloudflare Pages に接続します。

- 想定GitHubリポジトリ: `IFCSapp/mental-care-workbook-pwa`
- Build command: `npm run build`
- Output directory: `dist`
- Node.js: Cloudflare Pages の現行LTS相当
- 保護方針: Cloudflare Access で利用者を制限する想定

注意: Cloudflare Access は閲覧入口を守るための設定です。入力内容はサーバー保存せず、端末内保存と全体バックアップで扱います。

## visual-check

`npm run visual-check` を実行すると、開発サーバーを自動起動し、mobile / tablet / desktop 幅でスクリーンショットを保存します。あわせて console error、page error、リクエスト失敗、横スクロール、主要なカード・フォーム・ボタンの viewport 外はみ出しを軽く確認します。

結果は `reports/visual-check/` に出ます。このディレクトリは確認用の生成物なので、git 管理には含めません。

既にサーバーを起動している場合は、次のようにURLを指定できます。

```bash
npm run visual-check -- --url=http://localhost:5173
```

全ワークHTMLも対象にする場合は、次のように実行します。

```bash
npm run visual-check -- --include-works=true
```

- 既存の教材HTML（`public/works/`）はこの段階では改変しません。
- 各ワークは iframe で安全に起動します。

- GitHub公開リポジトリ作成、Cloudflare Pages接続、Cloudflare Access設定は別作業として扱います。

## Leaves UX 調整（2026-07-02）

### 方針

- 川背景は再生成せず、既存背景の上に薄いトーンフィルターをかけて刺激を下げる。
- 葉は主役ではなく、浮かんだ言葉を一時的に置くための媒体として扱う。
- 葉のサイズ・彩度・出現頻度を下げ、眺める余白を残す。
- 操作パネルは管理画面のように見えすぎないよう、川側となじむ柔らかい面にする。

### 変更内容

| ファイル | 変更内容 |
|---------|----------|
| `public/works/6.leaves_on_stream_single_full.html` | 川背景に薄いoverlay/filterを追加。葉の基準サイズを 420x180 → 220x95 に縮小。葉画像の彩度・コントラスト・不透明度を下げる。自動葉の間隔を 8秒 → 11秒 に延長。`止める` を `少し止める` に変更。入力後のガイドを「少しだけ、流れていく様子を眺めます。」へ一時更新。白くなりすぎないよう背景フィルターを青緑寄りに戻し、流れ筋・水面リップルを視認しやすく調整。左端の浅瀬/岸際から葉が出ないよう水面帯と出現位置を内側へ寄せ、葉の不自然な透明感を抑えた。 |
